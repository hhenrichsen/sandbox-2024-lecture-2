Database for Analytics

  • When you're starting off, it might make sense to use something off-the-shelf
  • Otherwise, a similar database to what you're already using can be good:
    • MySQL
    • PostgreSQL
    • MongoDB
  • Snowflake and similar products are used frequently in the industry

Fast Iteration

  • Learn what you can at the lowest cost tier
  • I personally use Postgres, Ktor, AlpineJS, and HTMX for my projects
  • Modern frameworks work well and give you a lot for free
  • Avoid Context Switching (so probably something fullstack)
  • If you don't have a favorite already, try a couple tools and pick one or two to pursue
  • Also, I like Tailwind / UnoCSS for prototyping (but not as much for production)
